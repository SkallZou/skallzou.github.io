Windows Event IDs

As a incident responder, it is important to know what is logged on the system, or what logs are important to trace back some malicious activities in a system. The Windows event log records specific events related to the system, security, and applications on a Microsoft system.

SYSTEM :


SECURITY :
  • 4624: An account was successfully logged on (LOCAL)
  • 4625: An account failed to log on (LOCAL)
  • 4702: A scheduled task was updated
  • 4662: An operation was performed on an object (DOMAIN)
  • 4768/4769: Authentication Kerberos has been requested (DOMAIN)
  • 4799: User groups enumeration by a process
  • 5140: A network share oject was access
  • 5379: Credential Manager credentials were read

    • Source: Service Control Manager
  • 7036: A service has changed to running state

APPLICATION :
    Source: ESENT (Database engine Windows)
  • 325: A new database is created by the database engine (NTDS.dit detection)
  • 327: The database engine detached a database (put offline the database, so cold analysis is possible)
